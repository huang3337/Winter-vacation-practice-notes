# 蓝桥杯嵌入式CT117E-M4实训平台学习



## 1. 点亮LED

1. PD2输出高电平（开启锁存器）;PC8-PC15输出低电平;PD2输出低电平（关闭锁存器，防止后续LCD通过相同引脚影响LED状态）。

   ```c
   HAL_GPIO_WritePin(GPIOD, GPIO_PIN_2, GPIO_PIN_SET);
   HAL_GPIO_WritePin(GPIOC, GPIO_PIN_8 , GPIO_PIN_RESET);
   HAL_GPIO_WritePin(GPIOD, GPIO_PIN_2, GPIO_PIN_RESET);
   ```

2. led一端输入高电平，GPIO口需输出低电平才可点亮led



## 2. 利用左移运算符实现点亮特定led

1. ```c
   void led_show(uint8_t led, uint8_t mode)
   {
   	HAL_GPIO_WritePin(GPIOD, GPIO_PIN_2, GPIO_PIN_SET);
   	if(mode)
   		HAL_GPIO_WritePin(GPIOC, GPIO_PIN_8 << (led - 1), GPIO_PIN_RESET);
   	else
   		HAL_GPIO_WritePin(GPIOC, GPIO_PIN_8 << (led - 1), GPIO_PIN_SET);
   	HAL_GPIO_WritePin(GPIOD, GPIO_PIN_2, GPIO_PIN_RESET);
   }
   ```

2. GPIO_PIN_8的二进制编码为0000 0001 0000 0000，

   GPIO_PIN_9的二进制编码为0000 0010 0000 0000，

   ……以此类推到GPIO_PIN_15。

   因此通过改变led的数使GPIO_PIN_8二进制编码左移成为相应的地址，达到控制特定GPIO口的目的。

## 5.LED与LCD引脚冲突问题

值得注意的是，本实验板的LCD与LED有引脚冲突，同样使用了PC8-PC15引脚。解决方法如下：

1. 由于cubemx中配置GPIO口时开启了锁存器，在初始化LCD的操作前添加以下代码，防止初始化LCD的操作影响LED的状态

```c
  HAL_GPIO_WritePin(GPIOD, GPIO_PIN_2, GPIO_PIN_RESET);
  //关闭锁存器，让后面的LCD操作不会改变LED的状态
  
  LCD_Init();//初始化LCD
  LCD_Clear(Black);//整块屏幕设定为。。。
  LCD_SetBackColor(Black);//显示文字背景为。。。
  LCD_SetTextColor(White);//设置文字颜色。。。
```

2. 后续主循环中同样可能有调整LCD的代码，为了不使LCD控制代码影响相应IO口对LED的控制，可以在每个会产生影响的函数定义中加入以下代码：

   ```c
   uint16_t temp = GPIOC->ODR;
   
   ……
   
   GPIOC->ODR = temp;
   ```

   如：

   ```c
   void LCD_Clear(u16 Color)
   {	
       u32 index = 0;
       LCD_SetCursor(0x00, 0x0000);
       LCD_WriteRAM_Prepare(); /* Prepare to write GRAM */
       for(index = 0; index < 76800; index++)
       {
           LCD_WriteRAM(Color);
       }
   }
   ```

   将会被改为：

   ```c
   void LCD_Clear(u16 Color)
   {	
       uint16_t temp = GPIOC->ODR;
       u32 index = 0;
       LCD_SetCursor(0x00, 0x0000);
       LCD_WriteRAM_Prepare(); /* Prepare to write GRAM */
       for(index = 0; index < 76800; index++)
       {
           LCD_WriteRAM(Color);
       }
   	GPIOC->ODR = temp;
   }
   ```

   这样达到在调用这类函数产生影响前暂时储存PC端的输出状态，在使用完函数后让PC端继承原先状态，保证每次调用LCD控制函数时都不会改变IO口原先的输出状态。

定时器时钟的中断频率：
$$
   f = \frac{f_{\text{system}}}{(ARR + 1)(PSC + 1)}
$$
定时器中断周期：
$$
T = \frac{1}{f} = \frac{(ARR + 1)(PSC + 1)}{f_{\text{system}}}
$$
   每隔 T 秒，进入一次定时器中断

   PSC不能超过2^16-1=65535

   系统频率**fsystem**为**80MHz**（8000 0000）

## 7. 利用计数器判断按键长按/短按

与【6】（1）中一样配置TIM3，但不用在NVIC Seting中使能TIM3中断；ARR也不用特意设置，保持原来最大值就行。

计数器CNT每次+1所耗费的时间为
$$
t_{\text{0}} = \frac{1}{f_{\text{0}}} = \frac{1}{\frac{f_{\text{system}}}{(PSC + 1)}}=\frac{(PSC + 1)}{f_{\text{system}}}
$$
而当CNT达到ARR+1时就会产生中断，若是直接利用CNT也可以得到时间，不需要利用ARR。

测CNT计数到10000的时间 **t** 可以为
$$
t = 10000*t_{\text{0}} = 10000*\frac{(PSC + 1)}{f_{\text{system}}}
$$
而系统频率**fsystem**为**80MHz**（8000 0000），则若是想让**t=1s**，则可以设置**PSC+1=8000**

使能函数是**start**

![](./%E5%B5%8C%E5%85%A5%E5%BC%8F%E8%93%9D%E6%A1%A5%E6%9D%AFCT117E-M4%E5%AE%9E%E8%AE%AD%E5%B9%B3%E5%8F%B0%E5%AD%A6%E4%B9%A0.assets/ScreenShot_2026-03-08_145326_163.png)

```c
uint8_t B1_state;
uint8_t B1_last_state = 1;//设置初值，否则设备上电后会直接进入判断逻辑
uint8_t B2_state;
uint8_t B2_last_state = 1;
void key_scan()
{
	B1_state = HAL_GPIO_ReadPin(GPIOB, GPIO_PIN_0);
	B2_state = HAL_GPIO_ReadPin(GPIOB, GPIO_PIN_1);
	
	if(B1_state == 0 && B1_last_state == 1) //按键B1按下
	{
		TIM3->CNT = 0;
	}
	else if(B1_state == 0 && B1_last_state == 0)//按键B1一直按着
	{
		if(TIM3->CNT >= 10000)  //按键B1长按1s
		{
			count++;
		}
	}
	else if(B1_state == 1 && B1_last_state == 0)//按键B1松开
	{
		if(TIM3->CNT < 10000)  //按键B1短按
		{
			count+=2;
		}
	}
	
	if(B2_state == 0 && B2_last_state == 1) //按键B2按下
	{
		TIM3->CNT = 0;
	}
	else if(B2_state == 0 && B2_last_state == 0)//按键B2一直按着
	{
		if(TIM3->CNT >= 10000) //按键B2长按1s
		{
			count--;
		}
	}
	else if(B2_state == 1 && B2_last_state == 0)//按键B2松开
	{
		if(TIM3->CNT < 10000)  //按键B2短按
		{
			count-=2;
		}
	}
	B1_last_state = B1_state;//更新状态
	B2_last_state = B2_state;
}
	
```

   ![](./%E5%B5%8C%E5%85%A5%E5%BC%8F%E8%93%9D%E6%A1%A5%E6%9D%AFCT117E-M4%E5%AE%9E%E8%AE%AD%E5%B9%B3%E5%8F%B0%E5%AD%A6%E4%B9%A0.assets/ScreenShot_2026-03-14_183557_496.png)

   为了捕获频率，则先获取周期T。关键在于输入捕获模式时，会在PWM波上升沿或者下降沿产生中断，而周期就是两个上升沿或者下降沿的时间间隔。

   方法：（用到的参数都是来自TIM17，也就是测量的定时器的）

   1. 在中断①设置**CNT=0**，中断②时用变量**capture_value=CNT**储存CNT的值

   2. 又已知：（t0是指每次计数增加所花的时间）
      $$
      t_{\text{0}} = \frac{1}{f_{\text{0}}} = \frac{1}{\frac{f_{\text{system}}}{(PSC + 1)}}=\frac{(PSC + 1)}{f_{\text{system}}}
      $$

   3. 则，两次中断间隔的时间周期，也就是图中PWM波的周期
      $$
      T = t_{\text{0}}*{capture\_value}
      $$

   4. 则，频率公式为：
      $$
      f = \frac{1}{T} = \frac{1}{t_{\text{0}}*{capture\_value}} = \frac{f_{\text{system}}}{(PSC + 1)*capture\_value}
      $$

4. 串口数据解析流程

   

   典型场景

   串口收到格式化字符串，如 `"CMD:MOVE,X:100,Y:200\r\n"`，需提取命令类型和参数。

   

   流程

   ```
   接收原始字节流
         ↓
   memset 清空缓冲区（防止脏数据干扰）
         ↓
   sscanf 按格式解析字符串，提取字段
         ↓
   strcmp 匹配命令类型，分支处理
   ```

   

   代码示例（具体示例可以看【13】）

   ```c
   #include <stdio.h>
   #include <string.h>
   
   void parse_serial(const char *raw) {
       char cmd[16];
       int x, y;
   
       // 1. 清空变量，避免上次残留数据
       memset(cmd, 0, sizeof(cmd));
   
       // 2. 按协议格式解析
       if (sscanf(raw, "CMD:%15[^,],X:%d,Y:%d", cmd, &x, &y) != 3) {
           // 解析失败，帧不完整或格式错误
           return;
       }
   
       // 3. 匹配命令，执行对应逻辑
       if (strcmp(cmd, "MOVE") == 0) {
           // 执行移动
       } else if (strcmp(cmd, "STOP") == 0) {
           // 执行停止
       }
   }
   ```

   

   三个函数说明（**注意参数类型**）

   | 函数     | 原型                                                | 作用                                             |
   | -------- | --------------------------------------------------- | ------------------------------------------------ |
   | `memset` | `void *memset(void *s, int c, size_t n)`            | 将内存块填充为指定值，常用于清零缓冲区           |
   | `sscanf` | `int sscanf(const char *str, const char *fmt, ...)` | 从字符串按格式提取数据，返回**成功匹配的字段数** |
   | `strcmp` | `int strcmp(const char *s1, const char *s2)`        | 比较两个字符串，相等**返回 0**                   |

   注意点

   - `sscanf` 的返回值**必须检查**，否则无法判断帧是否完整
   - `%15[^,]` 限制读取长度并以逗号为终止符，防止缓冲区溢出
   - `memset` 应在每帧解析前执行，不能只在初始化时调用一次
   - `strcmp` 返回值是整数，不能写成 `if (strcmp(...))` 来表示"相等"，逻辑相反

## 13. 利用定时器进行串口不定长数据接收

1. 原理

   **串口接收**：每次进入中断只能接收一个字节的数据

   **考虑错误情况**：

   1.每次接收一个字节就要判断该字节是否符合要求

   2.接收完所有字符，判断字符串是否符合要求

   **解决方法**：利用定时器，处理串口接收

   串口波特率=9600bit/s就是1s可以传输9600bit

   （比赛一般要求都是9600）

   串口传输一次数据包含起始位（1bit），数据位（8bit），结束位（1bit），一共10bit，10 * 1/9600 = 0.00104s =1.04ms

   **计算过程**：

   若波特率为9600bit/s，每次进入中断传输一个字节数据的时间就是1.04ms，那么在每次接收到数据进入中断时会将CNT设置为0，那么如果没有数据接收并进入中断就说明数据接收完毕，此时时间肯定超过1.04ms，即CNT超过某个值。

   若是将PCS+1设置为8000，CNT加一的时间就是8000/80MHz = 1/10000，CNT->15的时间就是15*1/10000 = 0.0015s = 1.5ms

   > （此处将CNT计数判断的时间设为1.5ms而不是1.04ms是为了方便计算且防止因为特殊原因导致计数混乱让数据丢失或者提前判断接收完成）

   ![](./%E5%B5%8C%E5%85%A5%E5%BC%8F%E8%93%9D%E6%A1%A5%E6%9D%AFCT117E-M4%E5%AE%9E%E8%AE%AD%E5%B9%B3%E5%8F%B0%E5%AD%A6%E4%B9%A0.assets/ScreenShot_2026-03-29_143339_544.png)

3. 代码逻辑

   利用计数器`CNT`来判断数据是否接受完成，完成则根据置1后的标志位（`rec_flag`）由**主循环**进入判断函数，由`sscanf`判断是否符合格式并将正确的数据写入变量，`strcmp`由写入变量相执行对应的指令，**格式正确且指令正确**就返回对应的值，否则串口返回**error**

   ps：`uart_data_rec();`要写在主循环；由于使用了定时器的计数器功能，要在主循环开始前对TIM4进行定时器使能，`HAL_TIM_Base_Start(&htim4);`

   ```c
   #include "headfile.h"
   
   uint8_t rec_data;
   char count,rec_flag;
   char send_buff[20], cmd[20],rec_buff[20];
   int x, y;
   void HAL_UART_RxCpltCallback(UART_HandleTypeDef *huart)
   {
   	if(huart->Instance == USART1)
   	{
   		//HAL_UART_Transmit(&huart1, &rec_data, 1, 50);
   		TIM4->CNT = 0;
   		rec_flag = 1;
   		rec_buff[count] = rec_data;
   		count++;
   		HAL_UART_Receive_IT(huart, &rec_data, 1);
   	}
   }
   void uart_data_rec(void)
   {
   	if(rec_flag)
   	{
   		if(TIM4->CNT > 15)//数据接收完成
   		{
   			
   			//按照协议对数据进行解析，如果格式不正确直接串口发送error并退出函数
   			if(sscanf(rec_buff, "CMD:%15[^,],X:%d,Y:%d", cmd, &x, &y) != 3)
   			{
   				sprintf(send_buff, "error!\r\n");
   				HAL_UART_Transmit(&huart1, (uint8_t *)send_buff, sizeof(send_buff), 50);
   				
   				//清空缓存
   				memset(rec_buff, 0, sizeof(rec_buff));
   				memset(send_buff, 0, sizeof(send_buff));
   				//还原标志位
   				rec_flag = 0;
   				count = 0;
   				return;
   			}
   			
   			//如果格式正确,开始匹配变量
   			if(strcmp(cmd, "MOVE") == 0)
   			{
   				//执行命令
   				sprintf(send_buff, "X:%d,Y:%d\r\n", x, y);
   				HAL_UART_Transmit(&huart1, (uint8_t *)send_buff, sizeof(send_buff), 50);
   			}else if(strcmp(cmd, "STOP") == 0)
   			{
   				//执行命令
   				sprintf(send_buff, "X:%d, Y:%d\r\n", x, y);
   				HAL_UART_Transmit(&huart1, (uint8_t *)send_buff, sizeof(send_buff), 50);
   			}else
   			{
   				sprintf(send_buff, "error!\r\n");
   				HAL_UART_Transmit(&huart1, (uint8_t *)send_buff, sizeof(send_buff), 50);
   			}
   			//清空缓存
   			memset(rec_buff, 0, sizeof(rec_buff));
   			memset(send_buff, 0, sizeof(send_buff));
   			//还原标志位
   			rec_flag = 0;
   			count = 0;
   		}
   	}
   }
   
   ```
