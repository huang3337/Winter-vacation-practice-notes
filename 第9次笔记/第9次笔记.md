# 第9次笔记

1. C51的sfr、sbit：

   sfr（special function register）：特殊功能寄存器声明	例：sfr P0 = 0x80;	声明P0口寄存器，物理地址为0x80

   sbit（special bit）：特殊位声明	例：sbit P0_1 = 0x81;    或    sbit P0_1 = P0^1;	声明P0寄存器的第1位

2. 74HC595

   74HC595是串行输入并行输出的移位寄存器，可用3根线输入串行数据，8根线输出并行数据，多片级联后，可输出16位、24位、32位等，常用于IO口扩展。

   ![](./%E7%AC%AC9%E6%AC%A1%E7%AC%94%E8%AE%B0.assets/ScreenShot_2026-03-01_143137_858.png)

   1. **数据设定**：设置 **SER** 电平。
   2. **移位**：给 **SERCLK** 一个上升沿，数据写入左侧寄存器一位。
   3. **循环**：重复步骤 1 和 2，直到 8 位数据全部就位。
   4. **锁存输出**：给 **RCLK** 一个上升沿，左侧数据复制到右侧，**QA-QH** 引脚电平更新。

3. LED点阵屏

   LED点阵屏的结构类似于数码管，只不过是数码管把每一列的像素以“8”字型排列而已LED点阵屏与数码管一样，有共阴和共阳两种接法，不同的接法对应的电路结构不同LED点阵屏需要进行逐行或逐列扫描，才能使所有LED同时显示

4. 可位寻址/不可位寻址

   | 特性                 | 可位寻址 (Bit-Addressable) | 不可位寻址 (Non-Bit-Addressable)               |
   | :------------------- | :------------------------- | :--------------------------------------------- |
   | **最小操作单位**     | 1个位（Bit）               | 1个字节（Byte，8个位）                         |
   | **是否有独立位地址** | 有                         | 无                                             |
   | **改变单一位的方法** | 直接赋值，如 `flag = 1;`   | 读-改-写，如 `REG \|= (1<<3);`                 |
   | **底层执行效率**     | 高（一条汇编指令解决）     | 略低（通常需要多条汇编指令）                   |
   | **对其他位的影响**   | 绝对不会干扰同字节的其他位 | 需要写代码时小心使用位运算，否则容易误改其他位 |

   理解这两个概念，主要是为了在对硬件（如寄存器配置、引脚控制）进行编程时，知道应该用简单的“直接赋值”（`=`），还是必须老老实实地用“位运算”（`&` , `|`，`^`）。

   寄存器不能全部使用可位寻址：1. 地址空间会“极速耗尽”；2. 硬件成本和芯片面积的剧增（走线太复杂）；3. 指令集膨胀（程序代码变大）；4. 绝大多数时候，我们“不需要”位寻址。
